1. O Pilar da Organização: Listas e Filas
Conceito: Como armazenar dados linearmente quando não sabemos o tamanho
final do conjunto.

Estrutura: Lista Ligada (Linked List)
   • O que é: Uma sequência de nós onde cada um aponta para o próximo.

   • Por que usar: Inserção e remoção rápidas sem precisar deslocar todos os
      outros elementos (como ocorre no Array).

Código (Baseado no Cormen - LIST-INSERT):

Plaintext
LIST-INSERT(L, x)
1 x.next = L.head
2 se L.head != NIL
3      então L.head.prev = x
4 L.head = x
5 x.prev = NIL




2. O Pilar da Eficiência: Árvores Binárias de Busca
(BST)
Conceito: Organizar dados de forma hierárquica para que a busca seja
extremamente veloz.

Estrutura: Árvore Binária
   • O que é: Cada "pai" tem dois "filhos". À esquerda ficam os menores, à direita
      os maiores.
   • Por que usar: Reduz o tempo de busca drasticamente. Em vez de olhar item
      por item, você descarta metade da árvore a cada passo.

Código (Baseado no Cormen - TREE-INSERT):

Plaintext
TREE-INSERT(T, z)
1 y = NIL
2 x = T.root
3 enquanto x != NIL
4      y = x
5      se z.key < x.key
6          então x = x.left
7          senão x = x.right
8 z.p = y
9 se y == NIL
10     então T.root = z
11     senão se z.key < y.key
12         então y.left = z
13         senão y.right = z




3. O Pilar da Prioridade: Heaps
Conceito: Gerenciar um conjunto de dados onde o elemento mais importante deve
estar sempre acessível.

Estrutura: Max-Heap (implementado em Array)
   • O que é: Uma árvore completa mapeada dentro de um vetor comum.

   • Por que usar: É a base para filas de prioridade (ex: triagem de hospital ou
      processos de CPU).

Código (Baseado no Cormen - MAX-HEAPIFY):

Plaintext
MAX-HEAPIFY(A, i)
1 l = LEFT(i)
2 r = RIGHT(i)
3 se l <= A.heap-size e A[l] > A[i]
4      então maior = l
5      senão maior = i
6 se r <= A.heap-size e A[r] > A[maior]
7      então maior = r
8 se maior != i
9      então trocar A[i] com A[maior]
10           MAX-HEAPIFY(A, maior)
4. O Pilar da Conexão: Grafos
Conceito: Representar relacionamentos complexos (redes sociais, rotas de entrega).

Estrutura: Lista de Adjacência
   • O que é: Um array onde cada índice representa um ponto (vértice) e contém
      uma lista de seus vizinhos.

   • Algoritmo de Estudo: BFS (Busca em Largura) para achar a menor distância.

Código (Baseado no Cormen - BFS):

Plaintext
BFS(G, s)
1 para cada vértice u em G.V - {s}
2      u.cor = BRANCO
3 s.cor = CINZA
4 Q = FILA-VAZIA()
5 ENQUEUE(Q, s)
6 enquanto Q não estiver vazia
7      u = DEQUEUE(Q)
8      para cada v em G.Adj[u]
9          se v.cor == BRANCO
10             v.cor = CINZA
11             ENQUEUE(Q, v)
12     u.cor = PRETO

5. Grafos (O Mapa do Mundo)
Como conectar pontos e achar o melhor caminho.

Busca em Largura (BFS)

Plaintext
BFS(G, s)
1 para cada vértice u ∈ G.V - {s}
2      u.cor = BRANCO; u.d = ∞; u.p = NIL
3 s.cor = CINZA; s.d = 0; s.p = NIL
4 Q = Ø
5 ENQUEUE(Q, s)
6 enquanto Q ≠ Ø
7      u = DEQUEUE(Q)
8      para cada v ∈ G.Adj[u]
9          se v.cor == BRANCO
10             v.cor = CINZA; v.d = u.d + 1; v.p = u
11             ENQUEUE(Q, v)
12     u.cor = PRETO

Algoritmo de Dijkstra (Caminho mais curto)

Plaintext
DIJKSTRA(G, w, s)
1 INITIALIZE-SINGLE-SOURCE(G, s)
2 S = Ø
3 Q = G.V
4 enquanto Q ≠ Ø
5      u = EXTRACT-MIN(Q)
6      S = S ∪ {u}
7      para cada vértice v ∈ G.Adj[u]
8          RELAX(u, v, w)




6. Programação Dinâmica (Nível Mestre)
Como não refazer trabalho que já foi feito. O exemplo clássico é o corte de hastes ou
a sequência de Fibonacci otimizada.

Corte de Haste (Bottom-up)

Plaintext
BOTTOM-UP-CUT-ROD(p, n)
1   seja r[0..n] um novo arranjo
2   r[0] = 0
3   para j = 1 até n
4       q = -∞
5       para i = 1 até j
6            q = max(q, p[i] + r[j - i])
7       r[j] = q
8   retorne r[n]

1. Algoritmos de Árvore Geradora Mínima (MST)
Essenciais para desenhar redes (elétricas, estradas ou computadores) com o menor

custo possível.
Algoritmo de Kruskal Ele utiliza uma estrutura chamada Sets (Conjuntos Disjuntos)

para evitar ciclos enquanto liga os pontos mais baratos.



Plaintext

MST-KRUSKAL(G, w)

1 A=Ø

2 para cada vértice v ∈ G.V

3   MAKE-SET(v)

4 ordenar as arestas de G.E em ordem não decrescente pelo peso w

5 para cada aresta (u, v) ∈ G.E (em ordem)

6   se FIND-SET(u) ≠ FIND-SET(v)

7      A = A ∪ {(u, v)}

8      UNION(u, v)

9 retorne A




2. Caminhos Mínimos com Pesos Negativos
O Dijkstra falha se houver custos negativos (como créditos ou perdas). O Bellman-

Ford resolve isso e ainda deteta "ciclos infinitos de perda".


Algoritmo de Bellman-Ford



Plaintext

BELLMAN-FORD(G, w, s)

1 INITIALIZE-SINGLE-SOURCE(G, s)

2 para i = 1 até |G.V| - 1

3   para cada aresta (u, v) ∈ G.E
4      RELAX(u, v, w)

5 para cada aresta (u, v) ∈ G.E

6    se v.d > u.d + w(u, v)

7      retorne FALSE // Ciclo negativo encontrado

8 retorne TRUE




3. Casamento de Padrões de Strings (KMP)
Fundamental para processar texto. O algoritmo KMP evita comparações inúteis,

tornando a busca muito mais rápida que o método "bruto".


Algoritmo de Knuth-Morris-Pratt (KMP)



Plaintext

KMP-MATCHER(T, P)

1 n = T.length

2 m = P.length

3 π = COMPUTE-PREFIX-FUNCTION(P)

4 q = 0 // número de caracteres casados

5 para i = 1 até n

6    enquanto q > 0 e P[q + 1] ≠ T[i]

7      q = π[q]

8    se P[q + 1] == T[i]

9      q=q+1

10   se q == m

11      imprimir "Padrão ocorre com deslocamento" i - m

12      q = π[q]

Algoritmo de Ford-Fulkerson



Plaintext

FORD-FULKERSON(G, s, t)

1 para cada aresta (u, v) ∈ G.E

2   (u, v).f = 0

3 enquanto existir um caminho aumentado p no grafo residual Gf

4   cf(p) = min {cf(u, v) : (u, v) está em p}

5   para cada aresta (u, v) em p

6      se (u, v) ∈ G.E

7           (u, v).f = (u, v).f + cf(p)

8      senão (v, u).f = (v, u).f - cf(p)

Plaintext

HUFFMAN(C)

2 Q = C // fila de prioridade mínima
